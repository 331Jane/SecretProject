# 内容展示网站

一个简洁美观的卡片式内容展示网站，支持动态添加卡片和图片上传。

## 功能特点

✨ **核心功能:**
- 📸 卡片式布局展示内容
- 🖼️ 支持上传图片（自动保存为 Base64）
- ➕ 动态添加和删除卡片
- 💾 数据自动保存到浏览器本地存储 **或** Firebase Cloud
- 📱 完全响应式设计，适配各种屏幕
- ☁️ **新增：Firebase 云同步** - 多人共享数据

### 两种使用模式

1. **本地模式** - 数据存储在浏览器本地（默认）
   - 无需配置，立即使用
   - 数据仅在本人浏览器保存
   
2. **云共享模式** - 集成 Firebase Firestore
   - 多人实时共享数据
   - 数据保存在云端
   - 任何人访问同一链接都能看到最新数据

## 如何使用

### 1. 打开网站
在 VS Code 中使用"Live Server"或其他本地服务器打开 `index.html`：
- 右键点击 `index.html` → "使用 Live Server 打开"
- 或手动在浏览器中打开：`http://localhost:5500` (端口号可能不同)

### 2. 添加卡片
1. 点击"添加新卡片"按钮
2. 填写标题和描述
3. （可选）上传图片
4. 点击"保存卡片"

### 3. 管理卡片
- 点击卡片下的"删除"按钮可以删除卡片
- 所有数据会自动保存到浏览器本地存储，刷新页面不会丢失

### 4. 配置 Firebase（可选）
如果想实现多人共享数据：

1. 点击右上角 **"⚙️ 配置"** 按钮
2. 按照 [FIREBASE_SETUP.md](FIREBASE_SETUP.md) 中的步骤配置
3. 配置完成后，所有数据会自动同步到云端
4. 分享链接给其他人，他们会看到相同的数据

详细步骤请参考：[Firebase 配置指南](FIREBASE_SETUP.md)

## 注意事项

### 关于图片存储
- 上传的图片会转换为 Base64 格式存储
- **本地模式**：图片存储在浏览器本地存储中（通常有 5-10MB 限制）
- **Firebase 模式**：建议使用 Firebase Storage 存储大型图片（需要额外配置）
- 当前版本所有图片都存储为 Base64，建议单个图片不超过 1MB

### 数据持久化

**本地模式：**
- 数据存储在浏览器的 `localStorage`
- 清除浏览器缓存会删除所有数据
- 不同浏览器/设备的数据独立

**Firebase 模式：**
- 数据存储在 Google Cloud Firestore
- 永久保存（只要 Firebase 项目存在）
- 所有访问同一 URL 的用户共享数据
- 需要网络连接
